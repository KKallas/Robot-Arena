# Robot Arena Automation Tools

**The Meta-Game:** We're building a robot battle arena where AI pilots fight AI hackers. But here's the twist—we're using AI agents to build it. LLMs managing GitHub issues, solving tasks, creating PRs. Robots building the arena for robots.

## Why This Matters

Robot Arena will run autonomous bot battles. No human pilots. No real-time control. Just uploaded scripts fighting it out while we watch.

So why not use the same approach for development?

- **Issues are tasks** → LLMs can parse structured tasks
- **Code is text** → LLMs can read and write it
- **PRs are diffs** → LLMs can create them
- **Reviews are checklists** → LLMs can verify them

We're getting an early start on the technology that will power the arena itself. Every tool we build here teaches us how autonomous agents behave, fail, and succeed.

## The Tools

### moderate_issues.py

Turns messy human issues into LLM-ready format.

```bash
# Preview issues that need formatting
python3 99-tools/moderate_issues.py --dry-run

# Test on specific issue (Claude runs, no GitHub changes)
python3 99-tools/moderate_issues.py --test --issue 123

# Live mode - formats issues and posts to GitHub
python3 99-tools/moderate_issues.py --issue 123
```

**What it does:**
1. Finds issues without `llm-ready` or `needs-human` labels
2. Sends issue to Claude with formatting instructions
3. Claude either asks clarifying questions or proposes formatted version
4. Once approved, adds `llm-ready` label

### solve_issues.py

Solves LLM-ready issues and creates PRs.

```bash
# Preview issues that would be solved
python3 99-tools/solve_issues.py --dry-run

# Test on specific issue (Claude runs, no push/PR)
python3 99-tools/solve_issues.py --test --issue 123

# Live mode - solves and creates PR
python3 99-tools/solve_issues.py --issue 123
```

**What it does:**
1. Finds issues with `llm-ready` label (no `in-progress`)
2. Claims issue (adds `in-progress` label)
3. Clones repo to temp directory
4. Sends issue to Claude with solving instructions
5. Validates changes against acceptance criteria
6. Commits, pushes, creates PR
7. Comments on issue with PR link

### fix_prs.py

Fixes PRs based on review comments.

```bash
# Preview PRs with review comments
python3 99-tools/fix_prs.py --dry-run

# Test on specific PR (Claude runs, no push)
python3 99-tools/fix_prs.py --test --pr 123

# Live mode - fixes and pushes
python3 99-tools/fix_prs.py --pr 123
```

**What it does:**
1. Finds open PRs with review comments (both inline and general)
2. Clones repo and checks out PR branch
3. **Phase 1 - Analysis:** Claude analyzes if feedback is actionable
   - If unclear: Posts clarification questions, removes `llm-ready` label (signals human attention needed)
   - If clear: Proceeds to Phase 2
4. **Phase 2 - Fix:** Claude makes the requested changes
5. Commits, pushes to PR branch
6. Replies to each comment: "Addressed in latest commit."

### contribute.sh (Wrapper)

Simple bash wrapper for the Python tools.

```bash
# Interactive mode selector
./contribute.sh

# Direct mode selection
./contribute.sh solve --dry-run
./contribute.sh moderate --test --issue 123
./contribute.sh fix --pr 123
```

## Three Testing Modes

| Mode | Claude Runs? | GitHub Changes? | Use Case |
|------|--------------|-----------------|----------|
| `--dry-run` | No | No | Preview what would happen |
| `--test` | Yes | No | Test Claude's behavior safely |
| (none) | Yes | Yes | Live execution |

## The Vision

This is practice for the real thing.

When Robot Arena launches, we'll have:
- **Bot Pilots** - AI scripts controlling arena bots
- **Bot Hackers** - AI scripts trying to compromise opponents
- **Match Orchestrator** - Autonomous system running battles
- **League Manager** - Autonomous system handling rankings, scheduling

Every issue these tools solve teaches us something about autonomous agents:
- How to structure tasks for machines
- How to validate machine work
- How to handle failures gracefully
- How to maintain human oversight without human bottlenecks

We're not just building tools. We're building the muscle memory for running an autonomous league.
